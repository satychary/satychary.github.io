The 'serverless' concept

A 'serverless' function still runs on a server (whose .com URL is not important, because we'd never use it directly!) - it is an anonymous function, with inputs, output.

A microservice seems similar - a URL that acts as a function. BUT, a microservice is itself composed of multiple functions (eg a TMDB lookup involves DB access, search...), each of which could be a serverless function.

Serverless: AWS

Amazon kicked off the concept, with AWS Lambda functions. The idea is to have developers access these from OTHER AWS (micro)services.
